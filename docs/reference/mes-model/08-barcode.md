# 08 — 条码与追溯体系

> **实体数：** 24 | **流程数：** 4  
> **覆盖：** 条码生成、条码状态管理、编码规则、容器管理

---

## 数据实体

### 编码规则
| 实体 | 说明 | 关键字段 |
|------|------|---------|
| 编码规则 | 各类编号的编码规则定义 | 规则编码、前缀、日期格式、流水号位数、校验位 |
| 编码算法 | 编码生成算法 | 算法编码、算法类型 |
| 编码段 | 已分配的编码段 | 编码规则、当前流水号、起始值、终止值 |

### 条码管理
| 实体 | 说明 | 关键字段 |
|------|------|---------|
| 条码 | 条码主数据（一物一码/一批一码） | 条码、物料编码、批次号、数量、状态、生产日期 |
| 条码范围 | 条码号段管理 | 起始条码、结束条码、用途 |
| 条码状态 | 条码流转状态字典 | 状态编码、状态名称（在库/已领用/已消耗/已报废等） |
| 条码日志 | 条码操作日志（追溯核心） | 条码、操作类型、操作人、时间、位置 |
| 标签 | 物理标签模板 | 标签编码、标签类型、打印模板 |

### 条码流转记录
| 实体 | 说明 |
|------|------|
| 已接收的条码数据 | 接收时扫描的条码记录 |
| 已入库的条码数据 | 入库完成的条码记录 |
| 已出库的条码 | 出库的条码记录 |
| 已上线的条码 | 投入产线的条码记录 |
| 已完工的条码 | 完工产品的条码记录 |
| 已退货的条码数据 | 退货的条码记录 |
| 已报废的条码 | 报废的条码记录 |
| 有缺陷的条码 | 发现缺陷的条码记录 |
| 待入库的条码数据 | 待入库的条码 |
| 待退货的条码数据 | 待退货的条码 |
| 待返修的条码数据 | 待返修处理的条码 |
| 已拣料的条码数据 | 已完成拣料的条码 |

### 容器
| 实体 | 说明 |
|------|------|
| 容器 | 料箱/托盘/工装板等容器 |
| 容器与物料关系 | 容器当前装载的物料及数量 |
| 清空的容器 | 已清空容器的历史记录 |

## 条码状态流转

```
打印/生成 ──→ 在库 ──→ 已拣料 ──→ 已出库/已上线
                  │                      │
                  │                      ├── 已完工（成品）
                  │                      └── 已消耗（原材料）
                  │
                  ├── 待检验 ──→ 合格 ──→ 在库
                  │              └── 不合格 ──→ 待退货
                  │
                  └── 待入库 ──→ 在库

任何状态都可能转入：
  有缺陷 ──→ 待返修 ──→ 在库（返修后）
           └── 已报废（无法返修）
```

## 追溯能力

```
正向追溯：条码 → 工单 → 工序 → 操作人 → 时间

反向追溯：成品条码 → 物料条码（用了哪些物料）
                  → 设备（在哪台设备生产）
                  → 工序（经手了哪些工序）
                  → 操作人（谁做的）
                  → 质检记录（每次检验的结果）
```
